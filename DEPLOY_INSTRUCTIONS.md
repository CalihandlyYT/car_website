# Инструкция по загрузке файлов на сервер

## Проблема: отсутствует requirements.txt на сервере

Файл `requirements.txt` был создан локально, но его нужно загрузить на сервер.

## Способы загрузки:

### Способ 1: Через FTP/SFTP клиент (FileZilla, WinSCP и т.д.)

1. Подключитесь к серверу через FTP/SFTP
2. Найдите папку, где находится ваш сайт (обычно `public_html`, `www`, или корневая папка проекта)
3. Загрузите файл `requirements.txt` в ту же папку, где находится `main.py`

### Способ 2: Через SSH (если есть доступ)

```bash
# Подключитесь к серверу
ssh username@your-server.com

# Перейдите в папку проекта
cd /path/to/your/project

# Создайте файл requirements.txt (или загрузите через scp)
nano requirements.txt
# Вставьте содержимое файла и сохраните (Ctrl+X, затем Y, затем Enter)
```

### Способ 3: Через панель управления хостингом

1. Войдите в панель управления хостингом (cPanel, Plesk и т.д.)
2. Откройте файловый менеджер
3. Найдите папку с вашим проектом
4. Загрузите файл `requirements.txt`

### Способ 4: Через Git (если используется)

```bash
# Добавьте файл в git
git add requirements.txt
git commit -m "Add requirements.txt"
git push origin main

# На сервере:
git pull origin main
```

## Проверка после загрузки:

После загрузки файла на сервер, установите зависимости:

```bash
# На сервере выполните:
pip install -r requirements.txt
```

Или если используется виртуальное окружение:

```bash
# Активируйте виртуальное окружение
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows

# Установите зависимости
pip install -r requirements.txt
```

## Важно:

- Убедитесь, что файл `requirements.txt` находится в той же папке, что и `main.py`
- Проверьте права доступа к файлу (должен быть доступен для чтения)
- После загрузки перезапустите приложение на сервере

## Содержимое requirements.txt:

```
Flask==3.0.0
Pillow>=10.0.0
```

## Если проблема сохраняется:

1. Проверьте путь к файлу на сервере
2. Убедитесь, что файл имеет правильное имя (requirements.txt, не requirements.txt.txt)
3. Проверьте права доступа к файлу
4. Попробуйте создать файл напрямую на сервере через SSH или панель управления
